CT2: Colorization Transformer via Color Tokens

Created Date: November 14, 2023

Colorization Transformer via Color Tokens (CT2) deal with incorrect semantic colors and undersaturation without any additional external priors.

The color tokens are used to formulate the colorization task as a classification problem to mitigate undersaturation. In previous works, regression loss yields under saturation because it outputs average saturation value from the sample.

  • A single-channel grayscale image is split into a sequence of patches
  • Image encoder (ViT) extracts features from them
  • Add positional embeddings to each patch

    • Color Encoder learns mapping from the input luminance channel L to the two associated color channels ab, which is performed in CIE Lab color space

  • take samples in the training set to calculate the empirical probability distribution of ab values in ab color space
  • divide the ab color space into 484 color patches, and calculate the mean probability distribution in each color patch
  • Among 484 color patches, encode 313 valid color patches by assigning a randomly initialized learnable vector to each patch in color embedding module

    • Below, embedded color patches are color tokens

  • add positional information into color tokens by applying conditional positional encodings. Because closer color tokens in quantized ab color space are, more similar they are in terms of color

    • Decoder

  • interact grayscale image features ZL with color features Ic, and finally generate colorful images
  • Luminance-selecting module classifies the image into 4 ranges of L. Then, discretizes the ab distribution into 313 one-hot encoded values and narrow the range of color candidates.
  • Color transformer has a color attention mechanism that utilizes color mask Mc
  • Upsampler expand the resolution of image patch tokens only
  • Color Query assign appropriate color and estimate the correct semantic color for each image pixel
