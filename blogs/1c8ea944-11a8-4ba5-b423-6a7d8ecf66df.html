Games on AWS 2023 학회 참가 후기

Created Date: October 24, 2023

오늘 코코네 Gnosis 팀원 분들과 판교 테크노밸리 (경기창조경제혁신센터) 에서 열린 Games on AWS 2023 학회를 다녀왔습니다. 제가 듣고 배운 내용을 요약한 것인데 중간에 제가 귀찮아져서 대충 요약한 점 고려하시고 읽어주시면 감사하겠습니다.


스케줄은 다음과 같았습니다


Keynote 세션에서는

- Games on AWS 소개: AWS 서울 region 이 아시아에서 다섯 번째로 생겨났고 한국에서 수 많은 유저들을 통해 돈을 많이 벌고 있으며 한국 일자리 창출 효과가 있고 및 고객을 위해 열심히 일하고 있다고 어필하였습니다.


- Second Dinner 소규모 개발사가 어떻게 SNAP 이라는 앱을 성공적으로 배포하였는지 설명하였습니다.

  • 처음에 5명으로 시작한 개발팀이 처음 게임 개발 후 50명으로 불어남
  • AWS Serverless 를 사용함으로써 Infra 관리 비용이나 인력, 시간을 절감하고 최고로 잘 할 수 있는 분야 (게임 개발에) laser focus 집중 하였습니다.
  • 또한 Production 배포가 전혀 어렵지 않음
  • IP 개발에 시간을 들이지 않고 Marvel IP 를 사용
  • 빠른 테스팅과 개선을 수 만번 반복하는 게 중요함

    • - Riot Games 도 AWS 의 도움으로 AWS serverless 로 Migration 잘 했다고 합니다

  • Riot Games 게임 자랑 동영상과 Worlds 2023 홍보 영상을 보여주었습니다.
  • AWS 에서 ESports 중개 분야 도움 받음

    • - Neowiz 는 2년 안에 AWS serverless 로 Migration 함

  • 장점: Latency 낮음, 보안 강화, migration 과정에서 오래된 코드 개선
  • 단점: Cloud 비용이 불어나서 꾸준한 비용 최적화가 필요
  • 팁: Migration 할 때 권한 설정 먼저, DB 이전을 맨 뒤에 함

    • Keynote 세션이 끝난 후 전시부스에 가서 설문조사 후에 에코백 다섯 개 받았습니다. 네오위즈 건물에 가서 점심을 받았는데 예상보다 더 푸짐했습니다.

    맛있게 먹고 주위를 둘러보다가 12시 30분에 있는 퀴즈 이벤트에서 EC2 가 뭐의 약자인지 문제 풀어서 담요 받았습니다. 줄 서서 기다리는 중에는 혹시 틀릴까 조마조마하게 있었는데 다행히 쉬운 문제 걸려서 쪽팔림 없이 퀴즈 상품으로 담요를 받았습니다. 그 후 다시 네오위즈 건물로 돌아가 트랙 3에 참석하였습니다.


    첫번째 강의: 데브시스터즈 - Amazon Sage Maker 로 게임 강화 하기


    지도학습은 모방이라는 한계가 있지만 강화학습은 목적 지향적으로 학습해서 더 낫다는 말씀을 하셨습니다. 강화 학습을 요약하자면: Agent --> action --> reward, Environment, State


    SageMaker 사용법

    ML 학습, 배포 End-to-End 로 훈련: S3 --> (ECR) cotaniner 에 코드 올림 --> 모델 S3에 올라감

    하이퍼파라미터 튜닝

  • Automatic model tuning AMT (serverless managed)
  • Sage maker --> Amazon EC2 instances (parameters) --> find most accurate

    • 최적의 파라미터 찾는 방법

  • Grid Search
  • Random Search
  • Bayesian optimization (과거의 경험을 기반으로)
  • Hyperband (limited resource 에서 효율적으로 찾고 가장 좋음. Learning curve 를 보고 성능이 낮은 훈련을 중단함)

    • Devsisters 에서 SageMaker를 사용한 이유

    쿠키런 퍼즐월드 스테이지 기획에는 난이도 조정과 맵 구성이 있는데 제작/수정 --> 테스트 --> 리뷰 --> 업데이트 반영 의 과정을 거칩니다.

    여기서 테스트 할 때 어려운 점은 신뢰성, 전문성이 필요하고 시간이 대략 수천 시간정도로 오래 걸린다는 것 입니다. 그래서 강화학습 퍼즐봇을 만들었다고 합니다. 퍼즐 스테이지 기획은 리텐션 (매출)에 큰 영향을 끼치는 만큼 해결방안이 필요해서 강화학습 퍼즐 플레이 봇을 만들어서 테스트 인력을 대체했다고 합니다.

    퍼즐봇을 통해 이루고자 하는 목표는

  • 세밀한 스테이지 난이도 설정
  • 기획 검증/ 분석
  • 신규 로직 및 기믹 에러 파악

    • 하지만 퍼즐봇 개발에 문제점은: On-Premise 개발 환경 세팅과, 성능 부족 개발과 프로덕셕에서 사용되어야 함 (scalable 하지 않다) 해결 방안으로 AWS 의 S3 ECR 과 SageMaker AMT 를 활용했습니다.


    SageMaker AMT 적용 과정

    Core 부분만 AWS SageMaker에 옯겨 hyperparameter 튜닝에 씀

    SageMaker 노트북 인스턴스 생성하고 AMT 노트북 코드 작성


    Estimator 세팅


    Hyperparameter Tuner 세팅 (여러 개의 Estimator 한번에 실행)


    결론적으로 학습 소요 시간 60% 감축했고 AWS 를 사용하면 컨시어지 기술 지원과 주말에 싸다는 장점이 있다고 합니다. 비용 효율화를 위해 Spot instances 와 절감형 plan을 썼다고 합니다.
